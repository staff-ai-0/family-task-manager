## Known gap flagged but deliberately NOT touched

The active `docker-compose.yml` runs backend with `uvicorn --reload` and `DEBUG=${DEBUG:-true}`, and frontend with `npm run dev --host` (Astro dev server with HMR). Both services bind-mount `./backend/app` and `./frontend` from the host into the containers, meaning a `git pull` effectively hot-swaps live code without a container restart. This is functional but is a **development configuration running in production**.

A proper hardening pass would need:

- A multi-stage frontend `Dockerfile` that runs `npm run build` at image build time and serves the built SSR entry (e.g. `node dist/server/entry.mjs`) at runtime.
- A `docker-compose.prod.yml` override flipping the backend command to `uvicorn --host 0.0.0.0 --port 8000` (no `--reload`), setting `DEBUG=false`, and dropping the host bind mounts.
- End-to-end smoke tests of the built SSR entry against auth middleware, i18n routing, env var propagation, and the budget pages.

This is a meaningful change with its own test risk, out of scope for today's deploy. Deferred as a separate follow-up.
