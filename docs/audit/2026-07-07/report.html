Audit · 2026-07-07

Family Task Manager — Launch Readiness & Competitive Intel

44-agent sweep: 11 repo audit dimensions (critical/high claims adversarially re-verified in code and on the prod host) + 7 web-research lanes covering 28 competitor products. 71 verified findings, 112 competitor features assessed.

Verdict

Engineering is ahead of the curve for this stage — tenant isolation, upload validation, webhook signatures, backups-on-a-timer, onboarding funnel, and bilingual UI all check out. What blocks a public launch is not code quality; it is three walls:

1 · Legal is at zero for a kids' app (no privacy policy or aviso de privacidad, children self-register with no consent flow, kid photos/chat flow to LLMs undisclosed, no way to delete or export a family). 2 · Billing punishes paying customers (one failed payment = instant silent downgrade with no recovery path; opening a checkout can clobber an active subscription). 3 · There is no safety net (no CI, no rollback, no offsite backup, no uptime alerting, Sentry wired but off).

The market news is very good: nobody serves Spanish-first LATAM, and nobody anywhere bundles chores + real money + budgeting + AI. The highest-ROI build is a "Family Bank" layer (payday + jars + parent-paid interest) — pure ledger math on tables you already have.

Part 1 · What blocks the launch

Legal — the hard wall 2 critical

No privacy policy, ToS, or aviso de privacidad anywhere Critical Effort M

Zero hits for privacy/términos/aviso across the frontend. LFPDPPP requires an aviso before collecting personal data; Google OAuth verification and PayPal live credentials also demand a published policy URL. Ship /privacidad + /terminos (ES/EN), link from register + footer, register the URL with Google and PayPal.

Children self-register with their own email — no age gate, no parental consent Critical Effort L

Anyone with a family join code registers at a self-chosen role (defaults CHILD) with immediate token issuance (auth.py:77-120). No birthdate field exists, so age verification is impossible. Fix: parent-initiated child creation or parent approval of join-code signups, optional child email, recorded consent (timestamp + consenting parent).

No family deletion, no data export — ARCO/GDPR rights unfulfillable High Effort M

FamilyService.delete_family exists but has zero route callers; parents are explicitly blocked from deleting their own account; only budget data is exportable. Needs a parent-initiated cascade delete + whole-family export.

Kid photos and chat flow to third-party LLMs with zero disclosure High Effort S

Task proof photos go to the vision model (default Gemini, per config.py:193); family chat is exposed to Jarvis via MCP. Disclose subprocessors in the aviso, gate AI-on-child-content behind a family-level parental opt-in, confirm zero-retention terms with providers.

Billing — what breaks with the first real customer 1 critical

Payment failure = instant silent downgrade, no recovery, no email Critical Effort M

The "3-day grace period" exists only in a docstring: payment_failed status is excluded from get_family_plan, past_due has zero writers, PAYMENT.SALE.COMPLETED (successful retry) is unhandled, and no dunning email exists. Customer's card hiccups once → family drops to Free mid-month, silently, permanently.

/checkout clobbers an active subscription before payment High Effort M

Opening (even abandoning) an upgrade checkout overwrites the active row with status='pending' and a new PayPal ID — paying customer downgraded, old PayPal sub orphaned and still billing (double-charge on completed upgrades).

No PayPal reconciliation; SUSPENDED/EXPIRED/refunds ignored; period_end never advances High Effort M

Webhook handles 3 event types; one missed webhook = permanent state drift. Add a daily re-sync via PayPalService.get_subscription in the existing sweep.

Also: join-code registration bypasses the family-member plan limit · zero billing emails · USD-only pricing for a Mexico-first product · trial invisible to the app

Safety net — nothing catches you 3 critical

No CI of any kind Critical Effort M

1,096 backend test functions + 26 Playwright specs, executed by nothing. Deploy script runs zero tests before recreating the prod pod. Minimal fix: one GitHub Actions workflow — pytest w/ postgres+redis services, astro build + astro check (already installed, never invoked).

No offsite backup — every copy lives on the disk it protects Critical Effort S

Daily 03:30 dumps verified live on 10.1.0.91, but on the same filesystem as the podman volumes (offsite push is a commented-out TODO in backup-db.sh). Uploads volume backed up nowhere. restore-db.sh defaults still point at the decommissioned GCP/Docker path; a restore drill has never been run. One rclone target + one drill closes the worst of it.

No uptime monitoring or alerting Critical Effort S

The good /ready endpoint (DB+Redis checks) has no consumer. Prod down at 2am = discovered by a user. One external health check (Cloudflare / Uptime Kuma on another host) + Sentry DSN in prod .env (code path already wired, DSN confirmed empty on the host) ≈ an afternoon.

Deploy has no rollback High Effort M

Untagged in-place image builds — if the new image fails health checks, prod is already down with nothing to fall back to. Tag builds with git SHA, keep previous tag, auto-revert on failed health check.

Performance & security — two systemic defects 1 critical

Sync OpenAI client blocks the event loop at 5 LLM call sites Critical Effort S

Jarvis (×2, in a tool loop up to 4 hops), calendar scanner, recipe importer, proof validator, category AI — all call the sync client inside async def on a 2-worker uvicorn. Two concurrent Jarvis chats freeze every request for up to 60s per hop; the app visibly degrades at tens of users. The fix already exists in the codebase: copy the receipt_scanner pattern (run_in_threadpool + timeout). Highest-leverage single fix in this report.

Rate limiting is spoofable + AI endpoints unmetered High ×2 Effort S

Uvicorn runs --forwarded-allow-ips="*", so a rotated X-Forwarded-For defeats every limiter (login brute-force, scan spam). And Jarvis chat/stream + calendar scan have no rate limit or plan metering at all — any Free account can run unbounded LLM spend. Key limits on CF-Connecting-IP, enable Redis storage, meter Jarvis via existing UsageTracking.

Also: zero security headers (HSTS/CSP/XFO) · SameSite unset on auth cookies · check-methods is an account-enumeration oracle

Product & polish gaps that face the customer

No support channel S

No email, form, or contact anywhere — for a paid product. Add soporte@ to footer, help, billing pages.

No operator admin L

Family lookup, subscription state, disable/delete — today it's raw psql on the prod box.

Core chore loop sends no push M

No TASK_ASSIGNED type, no "chores due today" morning reminder — while gigs, chat, pet all push. Highest-value notification in a chores app.

Notifications in random ES/EN mix M

9 services pass hardcoded literals; same kid gets both languages in one feed. Email service already has the _t() pattern — port it.

Anonymous locale flip-flops S

Landing defaults ES, login defaults EN, zero Accept-Language handling. Centralize in middleware: cookie → header → es.

Login page advertises credentials that don't exist S

parent@demo.com was never seeded (it's mom@demo.com). Every prospect who tries the demo bounces.

Email verification never enforced S

Flow exists; nothing reads the flag except resend. Gate email-triggering actions at minimum.

Landing CTAs point at /login S

"Comenzar gratis" → login instead of /register. One-line conversion fix. Plus: no 404/500 pages; budget first-account dead-end (a new family's first budget action has no way to create an account outside buried Settings).

Part 2 · Competitive intel — 28 products, 7 lanes

The white space is real, and it's yours Positioning

Every incumbent prices in USD; none publishes MXN pricing or serves Mexico (Greenlight/BusyKid/FamZoo/Acorns Early require US residency/SSN; Rooster is UK-only; the Spanish-native field is one solo-dev app with no money rails). And nobody in any market bundles chores + real money + full budgeting + AI — competitors are single-loop apps. LATAM fintech raised big in 2025 (Plata $250M, Klar $170M) but none targets family/kids money. The claim "la app de tareas, domingo y presupuesto familiar hecha para México" is uncontested.

Steal list — ranked by leverage on what you already have

1 · "Family Bank" bundle Adopt now Greenlight · BusyKid · FamZoo · Rooster

The four features every raved-about money app shares, all pure ledger math on cash_cents/points: payday (chores accrue all week, kid gets "paid" on a set day — the #1 reason parents stay), Save/Share/Spend jars with % auto-split of every payout (the three-jar pedagogy that actually teaches habits), parent-paid interest on the Save jar (kids viscerally learn compounding — Greenlight's entire upsell axis), and parent match ("I match 50% of what you save" — 401k psychology). A cron + a few cash_transaction inserts. This is the launch differentiator.

2 · Age-preset chore packs at onboarding Adopt now BusyKid

BusyKid's age-based preset chart ("set up in minutes") kills the cold-start problem — reviewers cite it constantly. Ship curated ES/MX template packs by age (or Jarvis generates them at family creation) pre-populating task templates + gig offerings with sensible point/MXN values. Same move fixes the empty-state cliff the product audit found. Include default Spanish rewards (tiempo de pantalla, salida al parque).

3 · Task mechanics parents pay for Adopt now Chorsee · S'moresUp · Sweepy · Modak · Thumsters

  • Sibling rotation on task templates — the "why always me" killer; it's why people pay Chorsee even though theirs is buggy. Your gig board already has rotation modes; extend to chores.
  • "X days since last completed" scheduling — Chorsee's angriest unmet request; real chores recur by elapsed time, not weekday.
  • Photo proof + approval on tasks — you already built the whole pipeline for gigs (uploads, auth proxy, AI pre-screen). Extend to task assignments.
  • Kid-proposed gigs — kid drafts an offering, parent approves the price. Kid agency; fits gig_offerings + HITL as-is.
  • 1-tap award/deduct with reason — capture spontaneous behavior ("helped his sister"); Thumsters takes 5 taps, beat them at 2. Or via Jarvis: "dale 5 puntos a Emma".

4 · Kiosk = the $630 Skylight, on a $150 tablet Adopt now Skylight · Hearth

Families pay $300–630 + $79/yr for a wall display; USD hardware pricing is prohibitive in Mexico. Your kiosk mode is 80% there. Polish into an always-on dashboard (week calendar + chores + meals + pet + leaderboard), add per-kid PIN switch (Chorsee's household-device model) and per-member colors everywhere (Hearth's most-loved detail). Market it: "convierte tu tablet en un Skylight." Also steal icon-based tap-through routines for pre-readers (morning/evening ordered checklists) — the single most-raved feature across both hardware products, and wiring routine completion to the pet makes yours the only one with a motivation layer.

5 · Kid engagement loops Joon · Habitica · Nipto · Thumsters Pet decision pending

  • Quest-mode kid UI (Joon, 4.7★): kid never sees a chore list — only quests that feed the pet; instant coin-drop on completion (ADHD parents cite immediacy as why it works). Your kid_pet + roles + optimistic-mutate patterns make this mostly frontend work — but it expands the pet, which you've flagged as uncertain. Decide before building.
  • Joon's failure mode is the warning: engagement decays weeks 4–8 once pet customization exhausts. Your two-currency economy (cash outlives cuteness for teens) + evolution stages are the countermeasures. ChoreMonster's orphaned audience went to Joon; your pet inherits the same appeal.
  • Weekly Family Cup (Nipto's loved loop): 7-day season, fresh start every Monday, winner picks a reward. Kiosk is the perfect leaderboard.
  • Family boss battle (Habitica's killer mechanic, defanged): weekly "Monstruo del Desorden" whose HP = the family's tasks; approvals deal damage. Cooperative, no anxiety-inducing HP loss.
  • Kid-visible goal jar (Thumsters/Greenlight): kid earmarks points toward a chosen reward with a progress jar. Kids check their bank constantly.
  • Star mode for ages 3–7 (Rooster): stars-not-points skin; families start earlier, stay longer.

6 · Budget module upgrades YNAB · Monarch · Copilot · Actual

  • Kid envelopes — no budget app on earth gives kids their own envelopes fed by chores. YNAB Together (6 seats) is the closest and users rave; yours would be unique. This is the bridge between your two products.
  • Month rollover + "cover overspending" flow — YNAB's behavioral core; your allocations lack it.
  • Bill calendar + 30/60-day forecast — Monarch's most-praised UX; you have recurring txns + a family calendar, just project them.
  • Learning categorization — Copilot's #1 raved feature; feed past corrections into the scanner prompt as few-shot, auto-suggest rules from repeats. Prompt work, not schema work.
  • Recurring-charge detection — "caught a forgotten subscription" is quantifiable value; very MX-relevant.
  • Don't build: bank aggregation (MX coverage poor — your receipt scanner + bank-email matcher IS the MX substitute), investment sync (Monarch's top complaint source), local-first (ship strong export instead).

7 · Packaging what's already built Cozi Max · Skylight Sidekick · Monarch AI

Three features competitors lead their marketing with, which you have and hide: AI flyer→calendar scan (Cozi Max's headline upsell; put a "escanea un aviso" button on kiosk + onboarding, tune on handwritten Spanish avisos escolares), AI meal planning (a Jarvis scheduled prompt — "plan dinners, add to shopping list" — is Cozi's whole Max tier), and Jarvis in budget pages (Monarch's marquee "AI Assistant" is a weaker version of what Jarvis+MCP already does — surface it with suggested Spanish money questions). Also: auto-post completions with proof photos into family chat with reactions (S'moresUp's "Campfire" liveliness, one event hook).

Cross-cutting lessons from their 1-star reviews

Reliability IS the feature

The #1 complaint across OurHome, Cozi, S'moresUp, FamilyWall, Nipto is sync bugs and notifications that never arrive. Breadth without reliability kills trust at 5M downloads. Your ops discipline is marketable.

Billing honesty is a trust weapon

Greenlight billing after cancellation, BusyKid refusing transfers, Thumsters locking paid users out — money-handling failures dominate fintech reviews. Self-serve cancel + transparent billing differentiates (and your dunning fix is prerequisite).

Fairness is the emotional trigger

Sweepy's effort-point gripes, Nipto's praised point-haggling: make point values negotiable ("propose new value", Jarvis arbitrates). Avoid shame mechanics — keep decay playful (pet), never punitive red bars.

Verification beats trust

BusyKid kids self-approved fake chores for months. Your photo-proof + AI pre-screen + parent approval already beats the category — market it.

Growth notes

  • Spanish SEO is near-empty and cheap to win — "cuánto domingo dar por edad", "tareas para niños recompensas", "TDAH rutinas". Every US player runs a content machine; nobody does it in Spanish.
  • Family-invite virality is intrinsic — make second-parent invite a first-run step; instrument it.
  • Distribution is app-store-first — no web-first success found in the category. PWA now; a Capacitor wrap (remote-URL shell + FCM/APNs) is post-launch, pre-growth work.
  • Joon proves clinical framing converts — Spanish content around rutinas/TDAH sustains premium pricing; unoccupied in LATAM. Later: B2B2C via Mexican banks (Greenlight distributes through 150+ banks now) and a colegio pilot program nobody has tried.

Part 3 · Suggested sequence

P0 — Don't launch without

~2–3 weeks
  • Legal: aviso de privacidad + ToS pages, consent checkbox, child-signup consent flow, family delete + export endpoints, AI disclosure/opt-in
  • Billing: grace period honored + SALE.COMPLETED recovery + dunning email; checkout guard; daily PayPal re-sync; join-code member-limit check
  • Safety net: offsite backup push + uploads tar + one restore drill; Sentry DSN set; external uptime check; minimal GitHub Actions CI; tagged images + rollback
  • Perf/sec: threadpool-wrap the 5 sync LLM sites; CF-Connecting-IP rate-limit key + Redis storage; meter Jarvis/calendar-scan; security headers
  • Customer floor: support email; task-assigned + morning-chores push; fix demo creds; locale middleware; notification i18n; CTAs → /register; 404/500 pages

P1 — Launch differentiators

~3–4 weeks
  • Family Bank: payday job + Save/Share/Spend jars + parent interest + match
  • Onboarding: age-preset ES/MX chore+gig+reward packs (Jarvis-generated); budget first-account empty state
  • Task mechanics: sibling rotation, interval-since-completion recurrence, photo proof on tasks, kid-proposed gigs, 1-tap points
  • Kid money: goal jar + kid envelopes in budget; star mode for little kids
  • Kiosk: per-kid PIN, member colors, leaderboard/routines panels; MXN PayPal plans + pricing page copy ("una membresía, toda la familia")
  • Packaging: scan-a-flyer + Jarvis meal-plan + Jarvis-in-budget surfaced in UI and landing copy

ongoing
  • Quest-mode kid UI + pet evolution stages — after the pet go/no-go decision
  • Family Cup weekly season; family boss battle; completion posts into chat
  • Budget: rollover/overspend UX, bill calendar + forecast, learning categorization, recurring detection
  • Capacitor wrap for app stores; referral program; Spanish SEO content machine; routines library; TDAH content angle
